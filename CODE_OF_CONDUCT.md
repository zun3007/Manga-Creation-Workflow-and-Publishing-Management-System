# Code of Conduct

## Our Standard

This project welcomes constructive participation from contributors, reviewers,
maintainers, students, and studio stakeholders. We expect participants to:

- Discuss ideas and code with respect, patience, and technical clarity.
- Critique the work rather than the person who produced it.
- Give actionable feedback and acknowledge reasonable trade-offs.
- Protect private studio information, personal data, and security reports.
- Respect role boundaries while making space for questions and disagreement.
- Take responsibility for mistakes and help repair their impact.

## Unacceptable Behavior

The following behavior is not acceptable:

- Harassment, threats, intimidation, stalking, or discriminatory language.
- Personal attacks, insults, sexualized content, or unwanted attention.
- Deliberate disruption, repeated bad-faith argument, or review retaliation.
- Publishing another person's private information without permission.
- Sharing credentials, private project data, or vulnerability details publicly.
- Using project access, status, or role authority to pressure or exclude others.

## Scope

This code applies in repository issues, pull requests, reviews, discussions,
project communication channels, demonstrations, and other spaces where someone
represents the project.

## Reporting Conduct Concerns

Report conduct concerns privately to the repository owner or maintainers through
the project's existing private team channel. Include the relevant links, dates,
participants, behavior, and any supporting evidence. Do not post sensitive
reports publicly or retaliate against a reporter.

Maintainers will limit report details to the people needed to assess and address
the concern. Conflicts of interest should be disclosed, and an uninvolved
maintainer should handle the report when practical.

## Enforcement

Maintainers may respond proportionately by providing a private correction,
removing content, issuing a warning, restricting participation, revoking project
access, or escalating a serious matter to the appropriate platform or authority.
Decisions should consider severity, repetition, impact, intent, and willingness
to repair harm.

Good-faith reporting is protected. Knowingly false reports, retaliation, and
attempts to expose a reporter are themselves violations of this code.
